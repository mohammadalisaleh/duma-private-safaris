1
Create Your GitHub Repository
1

Create a GitHub account (if you don't have one)

Go to github.com → click Sign Up → use a professional email like info@dumaprivatesafaris.com

2

Create a new repository

  • Click the + icon → New repository
  • Name it: duma-private-safaris
  • Set it to Public (required for free GitHub Pages)
  • Do NOT initialise with a README — leave it empty
  • Click Create repository
3

Upload all your files

On your new empty repo page, click uploading an existing file → drag and drop ALL the files from your download:

index.html 404.html sitemap.xml robots.txt CNAME _config.yml

Then click Commit changes (green button at the bottom).

2
Enable GitHub Pages
1

Go to repository Settings

In your repo, click the Settings tab (top right) → scroll down to Pages in the left sidebar.

2

Configure the source

  • Under Source, select Deploy from a branch
  • Branch: main  /  Folder: / (root)
  • Click Save
✓ Success: After 1–2 minutes, your site goes live at https://[your-username].github.io/duma-private-safaris/ — this is temporary. Your real domain comes in Step 3.
3

Add your custom domain in GitHub Pages

  • Still in Settings → Pages, find Custom domain
  • Type: www.dumaprivatesafaris.com
  • Click Save
  • Tick Enforce HTTPS (may take a few minutes to appear)
Note: The CNAME file you uploaded already contains your domain — GitHub uses this automatically. The Settings UI and the CNAME file work together.
3
Connect Your Domain via Cloudflare
Before you start: You need to own the domain dumaprivatesafaris.com. If you haven't bought it yet, purchase it from Namecheap, GoDaddy, or directly through Cloudflare Registrar (cheapest option — no markup).
1

Create a Cloudflare account

Go to cloudflare.com → Sign Up → Free plan is sufficient to start.

2

Add your website to Cloudflare

  • Click Add a Site → enter dumaprivatesafaris.com
  • Choose the Free plan → Continue
  • Cloudflare will scan your existing DNS records — click Continue
3

Add these DNS records in Cloudflare

Go to DNS → Records → Add the following records exactly:

TypeNameContent / ValueProxyTTL
A@185.199.108.153✓ ProxiedAuto
A@185.199.109.153✓ ProxiedAuto
A@185.199.110.153✓ ProxiedAuto
A@185.199.111.153✓ ProxiedAuto
CNAMEwww[username].github.io✓ ProxiedAuto

Replace [username] with your actual GitHub username.

Why 4 A records? GitHub Pages runs on 4 IP addresses for redundancy and speed. All four are required.
4

Point your domain's nameservers to Cloudflare

Cloudflare will show you two nameservers, something like:

ada.ns.cloudflare.com bob.ns.cloudflare.com

Go to wherever you registered your domain (Namecheap, GoDaddy etc.) → find Nameservers → replace the existing ones with the Cloudflare nameservers above. This change takes 15 minutes to 24 hours to propagate globally.

5

Enable these Cloudflare settings for maximum SEO & speed

  • SSL/TLS → Overview: Set to Full (Strict)
  • SSL/TLS → Edge Certificates: Enable Always Use HTTPS
  • SSL/TLS → Edge Certificates: Enable HTTP Strict Transport Security (HSTS)
  • Speed → Optimization: Enable Auto Minify (JS, CSS, HTML)
  • Speed → Optimization: Enable Brotli compression
  • Caching → Configuration: Set Browser Cache TTL to 1 month
  • Rules → Page Rules: Add a redirect from dumaprivatesafaris.com/* → https://www.dumaprivatesafaris.com/$1 (301 redirect)
✓ Why this matters for SEO: HTTPS is a Google ranking signal. Brotli compression and minification reduce page load time, which directly improves your Core Web Vitals score — a key ranking factor since 2021.
4
Register with Google & Bing
1

Submit to Google Search Console

  • Go to search.google.com/search-console
  • Click Add Property → choose Domain
  • Enter dumaprivatesafaris.com
  • Verify ownership via DNS record — Cloudflare makes this easy: copy the TXT record Google gives you → paste it into Cloudflare DNS as a new TXT record → click Verify
  • Go to Sitemaps → submit: https://www.dumaprivatesafaris.com/sitemap.xml
2

Submit to Bing Webmaster Tools

Go to bing.com/webmasters → Add your site → import from Google Search Console (one click) → Bing will also import your sitemap automatically.

Why Bing? Bing powers DuckDuckGo and Yahoo search. Collectively they represent ~15–25% of search traffic in the US and UK — your primary markets.
3

Test your structured data / schema

Go to search.google.com/test/rich-results → paste your URL → verify that Google detects:

  • FAQPage High impact
  • Product (safari packages) High impact
  • Organization / LocalBusiness Medium impact
  • BreadcrumbList Useful
4

Set up Google Analytics 4

  • Go to analytics.google.com → create an account → create a Property
  • Choose Web → enter your URL
  • Copy the G-XXXXXXXXXX measurement ID
  • Add this snippet just before </head> in your index.html:
<!-- Google Analytics 4 --> <script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"></script> <script> window.dataLayer = window.dataLayer || []; function gtag(){dataLayer.push(arguments);} gtag('js', new Date()); gtag('config', 'G-XXXXXXXXXX'); </script>

Off-Page SEO — Push to the Globe

These steps are done outside your website but are critical for ranking globally. Do them in order — the first three are the highest impact.

1

Google Business Profile Highest impact

Go to business.google.com → create a profile for Duma Private Safaris → add your address (Arusha, Tanzania), phone, website, photos and category (Safari Tour Agency). This is what makes you appear in Google Maps and local search results worldwide.

2

Safari & Travel Directories Highest impact

Each listing is a backlink — a vote of authority that boosts your Google ranking. Submit your company to:

  • safaribookings.com — largest safari-specific directory
  • tripadvisor.com — create a business listing
  • lonelyplanet.com — submit your operator
  • tanzaniatouristboard.go.tz — official Tanzania tourism registry
  • arusha-info.com — Arusha tourism directory
  • responsibletravel.com — luxury ethical travel
3

Social Media Profiles Medium impact

Create and fully fill out profiles on Instagram, Facebook and LinkedIn — make sure your website URL is in every profile bio. These are treated as trust signals by Google.

4

Update CNAME and sitemap with real domain

Once your domain is confirmed, open sitemap.xml and replace all instances of dumaprivatesafaris.com with your actual domain if it differs. Same for the schema markup inside index.html.

5
